M3 — RDDs & DataFrames

Los dos modelos de datos de Spark. RDD es el nivel bajo (control total), DataFrame es el nivel alto (optimización automática). Aprende cuándo, cómo y por qué usar cada uno.

1. RDD — Resilient Distributed Dataset

El RDD es la abstracción fundamental de Spark. Un RDD es una colección de objetos Python distribuida en particiones a través del cluster, inmutable y con linaje (sabe cómo reconstruirse si una partición falla).

CaracterísticaDetalle
ResilienteSi se pierde una partición, Spark la recalcula desde su linaje (grafo de transformaciones, no replica datos)
DistribuidoDividido en particiones repartidas entre executors del cluster
DatasetColección de cualquier tipo Python: tuplas, dicts, objetos, strings…
LazyLas transformaciones no se ejecutan hasta que se llama a una acción
Sin schemaEl RDD no sabe qué hay dentro — es responsabilidad del programador

1.1 Crear RDDs

python
from pyspark.sql import SparkSession

spark = SparkSession.builder.master("local[*]").appName("m3-rdd").getOrCreate()
sc = spark.sparkContext  # SparkContext: acceso al nivel RDD

# ── Desde colección Python (parallelizar)
rdd1 = sc.parallelize([1, 2, 3, 4, 5, 6, 7, 8], numSlices=4)
print(rdd1.getNumPartitions())  # 4

# ── Desde fichero de texto (cada línea = un elemento)
# rdd2 = sc.textFile("/path/al/fichero.txt")
# rdd3 = sc.textFile("s3a://mi-bucket/datos/*.csv")

# ── Desde fichero de pares clave-valor
# rdd_pares = sc.sequenceFile("/path/sequence")

# ── Desde otro RDD (transformación)
rdd_cuadrados = rdd1.map(lambda x: x ** 2)  # todavía no ejecuta nada

# Ver los datos (acción → fuerza ejecución)
print(rdd_cuadrados.collect())  # [1, 4, 9, 16, 25, 36, 49, 64]

1.2 Transformaciones de RDD (lazy)

python
datos = sc.parallelize(range(1, 11))

# map: transformar cada elemento
pares_rdd = datos.map(lambda x: (x, x % 2 == 0))  # (valor, es_par)

# filter: seleccionar elementos
solo_pares = datos.filter(lambda x: x % 2 == 0)

# flatMap: cada elemento puede producir VARIOS (o cero) registros
frases = sc.parallelize(["hola mundo", "spark es potente", "rdd"])
palabras = frases.flatMap(lambda frase: frase.split(" "))
print(palabras.collect())  # ['hola', 'mundo', 'spark', 'es', 'potente', 'rdd']

# distinct: eliminar duplicados
con_dupes = sc.parallelize([1, 2, 2, 3, 3, 3])
sin_dupes = con_dupes.distinct()

# union: combinar dos RDDs
rdd_a = sc.parallelize([1, 2, 3])
rdd_b = sc.parallelize([3, 4, 5])
union_rdd = rdd_a.union(rdd_b)  # [1, 2, 3, 3, 4, 5]: NO elimina duplicados

# mapPartitions: función se aplica a la partición entera (eficiente para init de conexiones)
def procesar_particion(iterador):
    # Aquí podrías inicializar conexión DB una vez por partición
    return (x * 2 for x in iterador)

dobles = datos.mapPartitions(procesar_particion)

# sortBy: reordenar
descendente = datos.sortBy(lambda x: -x)
print(descendente.take(3))  # [10, 9, 8]

1.3 RDDs de pares clave-valor (PairRDD)

python
# Un PairRDD es un RDD de tuplas (clave, valor)
ventas = sc.parallelize([
    ("Norte", 1200), ("Sur", 850), ("Norte", 600),
    ("Este", 400),   ("Sur", 920), ("Norte", 780)
])

# reduceByKey: reduce por clave (más eficiente que groupByKey)
total_por_zona = ventas.reduceByKey(lambda a, b: a + b)
print(total_por_zona.collect())
# [('Norte', 2580), ('Sur', 1770), ('Este', 400)]

# groupByKey: agrupa valores por clave (usa más memoria)
grupado = ventas.groupByKey()  # cada valor es un iterable
grupado_list = grupado.mapValues(list)
print(grupado_list.collect())
# [('Norte', [1200, 600, 780]), ('Sur', [850, 920]), ('Este', [400])]

# sortByKey: ordenar por clave
ordenado = total_por_zona.sortByKey()
print(ordenado.collect())  # [('Este', 400), ('Norte', 2580), ('Sur', 1770)]

# join: unir dos PairRDDs por clave
nombres = sc.parallelize([("Norte", "Zona Norte"), ("Sur", "Zona Sur")])
join_result = total_por_zona.join(nombres)
print(join_result.collect())
# [('Norte', (2580, 'Zona Norte')), ('Sur', (1770, 'Zona Sur'))]

# mapValues: transformar solo el valor
promedio = ventas.groupByKey().mapValues(lambda vals: sum(vals) / len(list(vals)))

1.4 Acciones de RDD (fuerzan ejecución)

python
rdd = sc.parallelize(range(1, 101))

collect()           # traer TODO al driver — ⚠️ solo para datos pequeños
take(n)             # traer los primeros n elementos
first()             # primer elemento
count()             # número de elementos
countByValue()      # {valor: frecuencia}
sum()               # suma (solo numérico)
min()               # mínimo
max()               # máximo
mean()              # media
reduce(f)           # reducir con función binaria
fold(cero, f)       # reduce con valor cero inicial por partición
aggregate(cero, seqOp, combOp)  # reducción con tipo de retorno diferente

# saveAsTextFile: guardar resultado
rdd.map(str).saveAsTextFile("/tmp/resultado_rdd")

# foreach: ejecutar acción en cada elemento (sin retorno al driver)
rdd.foreach(lambda x: print(x))  # ejecuta en executors

# Ejemplo práctico
print(rdd.count())       # 100
print(rdd.sum())         # 5050
print(rdd.take(5))       # [1, 2, 3, 4, 5]
print(rdd.reduce(lambda a, b: a + b))  # 5050

1.5 Persistencia / Caché

python
from pyspark import StorageLevel

rdd_costoso = (
    sc.textFile("/datos/grande.csv")
      .flatMap(lambda l: l.split(","))
      .map(str.strip)
      .filter(bool)
)

# cache() = MEMORY_AND_DISK por defecto en DataFrames, MEMORY_ONLY en RDDs
rdd_costoso.cache()

# Niveles de almacenamiento
rdd_costoso.persist(StorageLevel.MEMORY_ONLY)         # solo RAM (rápido, riesgo OOM)
rdd_costoso.persist(StorageLevel.MEMORY_AND_DISK)     # RAM, si no cabe → disco
rdd_costoso.persist(StorageLevel.DISK_ONLY)           # solo disco
rdd_costoso.persist(StorageLevel.MEMORY_AND_DISK_2)   # replicado en 2 nodos

# Primeras acciones después de persist = ejecuta + almacena en caché
print(rdd_costoso.count())  # ejecuta + cachea
print(rdd_costoso.take(5))  # usa caché — no re-ejecuta

# Liberar caché cuando ya no se necesita
rdd_costoso.unpersist()

2. DataFrames: RDDs con esquema

Un DataFrame es un RDD de Row con un schema conocido (nombres y tipos de columnas). El optimizador Catalyst usa el schema para generar planes físicos eficientes. Usa DataFrames por encima de RDDs para casi todo.

RDDDataFrame
SchemaNo (cualquier objeto Python)Sí (StructType con tipos)
OptimizaciónNinguna (caja negra para Spark)Catalyst optimizer + Tungsten
RendimientoLínea baseNormalmente 5-10× más rápido
APImap/filter/reduce funcionalselect/filter/groupBy/join declarativa
SQLNoSí (createOrReplaceTempView)
Cuándo usar RDDDatos no estructurados, algoritmos que necesitan control total de particiones, operaciones que el DataFrame no puede expresar

2.1 Crear DataFrames

python
from pyspark.sql import SparkSession
from pyspark.sql.types import (
    StructType, StructField,
    StringType, IntegerType, DoubleType, BooleanType, TimestampType
)

spark = SparkSession.builder.master("local[*]").appName("m3-df").getOrCreate()

# ── Desde lista de dicts (inferir schema)
datos = [
    {"nombre": "Alice", "salario": 42000, "dpto": "Ingeniería", "activo": True},
    {"nombre": "Bob",   "salario": 35000, "dpto": "Marketing",  "activo": True},
    {"nombre": "Clara", "salario": 48000, "dpto": "Ingeniería", "activo": False},

2.3 Transformaciones: select y withColumn

python
from pyspark.sql import functions as F
from pyspark.sql.functions import col, lit, when, round as spark_round

# ── select: elegir/transformar columnas
df.select("nombre", "salario").show()
df.select(col("nombre"), col("salario") * 1.1).show()
df.select("nombre", (col("salario") * 1.1).alias("salario_nuevo")).show()
df.select(F.upper(col("dpto")).alias("dpto_norm")).show()

# ── withColumn: añadir o reemplazar una columna
df2 = df.withColumn("salario_neto", spark_round(col("salario") * 0.78, 2))
df3 = df2.withColumn("nivel",
    when(col("salario") >= 45000, "senior")
   .when(col("salario") >= 38000, "mid")
   .otherwise("junior"))

# ── withColumnRenamed: renombrar
df4 = df3.withColumnRenamed("nombre", "empleado_nombre")

# ── drop: eliminar columnas
df5 = df4.drop("activo", "nivel")

# ── cast: cambiar tipo
df6 = df.withColumn("salario", col("salario").cast(DoubleType()))

# Encadenar transformaciones
resultado = (df
    .withColumn("salario_neto", spark_round(col("salario") * 0.78, 2))
    .withColumn("es_senior", col("salario") >= 45000)
    .withColumn("dpto_norm", F.upper(F.trim(col("dpto"))))
    .select("nombre", "dpto_norm", "salario", "salario_neto", "es_senior")
)
resultado.show()

2.4 Transformaciones: filter, distinct, sort, limit

python
# ── filter / where (sinónimos)
ing = df.filter(col("dpto") == "Ingeniería")
ing = df.where("dpto = 'Ingeniería'")  # también acepta SQL string

# Múltiples condiciones: & (AND), | (OR), ~ (NOT)
activos_senior = df.filter((col("activo") == True) & (col("salario") >= 45000))

# IN, LIKE, isNull, isNotNull, between
df.filter(col("dpto").isin("Ingeniería", "Marketing"))
df.filter(col("nombre").like("A%"))
df.filter(col("salario").between(35000, 50000))
df.filter(col("dpto").isNotNull())

# ── sort / orderBy (sinónimos)
df.sort("salario").show()               # ascendente
df.sort(col("salario").desc()).show()   # descendente
df.orderBy(["dpto", col("salario").desc()])  # multi-columna

# ── distinct + dropDuplicates
df.distinct()  # filas únicas (todas las columnas)
df.dropDuplicates(["dpto"])  # primer registro por dpto (no determinista)
df.dropDuplicates(["dpto", "nombre"])

# ── limit
df.limit(10).show()  # solo 10 filas en el resultado final

2.5 Aggregaciones: groupBy y agg

python
from pyspark.sql.functions import (
    count, sum, avg, min, max,
    collect_list, collect_set,
    first, last, stddev, variance,
    countDistinct
)

# Operaciones globales (sin groupBy)
df.agg(avg("salario").alias("media_global"),
       max("salario").alias("max_global")).show()

# groupBy + agg
resumen = (df.groupBy("dpto")
    .agg(
        count("*").alias("n_empleados"),
        avg("salario").alias("salario_medio"),
        max("salario").alias("salario_max"),
        min("salario").alias("salario_min"),
        stddev("salario").alias("desv_std"),
        collect_list("nombre").alias("nombres"),
        countDistinct("activo").alias("estados_distintos")
    )
    .orderBy(col("salario_medio").desc())
)
resumen.show(truncate=False)

# Pivot: convertir filas en columnas
pivot = (df.groupBy("nombre")
    .pivot("dpto")  # crea una columna por cada dpto
    .agg(avg("salario"))
)
pivot.show()

2.6 Joins

python
dptos = spark.createDataFrame([
    {"dpto": "Ingeniería", "edificio": "A", "presupuesto": 500000},
    {"dpto": "Marketing",  "edificio": "B", "presupuesto": 250000},
    {"dpto": "RRHH",       "edificio": "A", "presupuesto": 120000},
])

# inner join (defecto)
df.join(dptos, on="dpto", how="inner").show()

# Tipos de join
# inner, left, right, full (outer), semi, anti, cross

# left join: todos los empleados, aunque no haya dpto
df.join(dptos, on="dpto", how="left").show()

# anti join: empleados SIN departamento coincidente
df.join(dptos, on="dpto", how="anti").show()

# join con condición más compleja
df.join(dptos,
    (df.dpto == dptos.dpto) & (df.salario < dptos.presupuesto * 0.01),
    how="inner").show()

# ⚠️ COLUMNAS DUPLICADAS: si ambos DF tienen columas con el mismo nombre
# usar alias en los DataFrames
df.alias("emp").join(
    dptos.alias("dep"),
    col("emp.dpto") == col("dep.dpto")
).select("emp.nombre", "dep.edificio").show()

# Broadcast join: cuando uno es pequeño (<10MB) → evita shuffle
from pyspark.sql.functions import broadcast
df.join(broadcast(dptos), on="dpto").show()  # Hint para Catalyst
