Machine Learning distribuido con Spark. Pipelines, transformers, estimators, validación cruzada y modelos desde clasificación hasta clustering a escala de petabytes.

1. Arquitectura de MLlib: Transformer, Estimator, Pipeline

MLlib se estructura en tres conceptos que reflejan el ciclo real de ML: preparar datos, entrenar modelo, predecir.

Concepto¿Qué hace?Ejemplo
TransformerTransforma un DataFrame en otro. Tiene método .transform(df). Ya está "configurado" — no aprende de datos.VectorAssembler, Tokenizer, modelo entrenado
EstimatorAprende de datos. Tiene método .fit(df) que devuelve un Transformer (modelo).RandomForestClassifier, StandardScaler
PipelineSecuencia de Transformers y Estimators. Es un Estimator que al hacer .fit() ejecuta toda la cadena.Pipeline(stages=[scaler, clf])
PipelineModelPipeline ya entrenado — solo Transformers. Listo para predecir.Resultado de pipeline.fit(df_train)
python
from pyspark.ml import Pipeline
from pyspark.ml.feature import (
    VectorAssembler, StringIndexer, OneHotEncoder, StandardScaler
)
from pyspark.ml.classification import RandomForestClassifier
from pyspark.ml.evaluation import BinaryClassificationEvaluator
from pyspark.sql import SparkSession
from pyspark.sql.functions import col

spark = SparkSession.builder.master("local[*]").appName("m5-mllib").getOrCreate()

# Dataset de ejemplo: predicción de churn
datos = spark.createDataFrame([
    (1, "ES", 24, 850.0, 3, 0),
    (2, "FR", 35, 1500.0, 8, 0),
    (3, "DE", 28, 200.0,  1, 1),   # churn=1
    (4, "ES", 45, 4500.0, 15, 0),
    (5, "FR", 22, 100.0,  0, 1),   # churn=1
    (6, "DE", 38, 2200.0, 9, 0),
    (7, "ES", 29, 300.0,  2, 1),
    (8, "FR", 52, 5000.0, 20, 0),
], ["id", "pais", "edad", "ltv", "meses", "churn"])

# Train/test split
df_train, df_test = datos.randomSplit([0.75, 0.25], seed=42)
print(f"Train: {df_train.count()}, Test: {df_test.count()}")

2. Feature Engineering: preparar los datos

python
# ── StringIndexer: string → índice numérico (0, 1, 2...)
#    Necesario porque los modelos de Spark solo aceptan números
indexer = StringIndexer(
    inputCol="pais",
    outputCol="pais_idx",
    handleInvalid="keep"   # "error", "keep" (categoría extra), "skip"
)

# ── OneHotEncoder: índice → vector binario (0/1)
#    [0, 1, 2] → [[1,0,0], [0,1,0], [0,0,1]]
encoder = OneHotEncoder(inputCols=["pais_idx"], outputCols=["pais_ohe"])

# ── VectorAssembler: juntar todas las features en un Vector
assembler = VectorAssembler(
    inputCols=["pais_ohe", "edad", "ltv", "meses"],
    outputCol="features_raw",
    handleInvalid="skip"
)

# ── StandardScaler: normalizar a media=0, std=1
scaler = StandardScaler(
    inputCol="features_raw",
    outputCol="features",
    withMean=True,   # centrar en 0
    withStd=True     # escalar a std=1
)

# Sin Pipeline: aplicar manualmente (para entender qué hace cada paso)
modelo_indexer = indexer.fit(df_train)
df_idx = modelo_indexer.transform(df_train)
df_idx.select("pais", "pais_idx").show(6)
# +----+--------+
# |pais|pais_idx|
# +----+--------+
# |  ES|     0.0|
# |  FR|     1.0|
# |  DE|     2.0|

3. Pipeline completo: clasificación binaria

python
# ── Renombrar la etiqueta (los clasificadores esperan columna "label")
df_train_ml = df_train.withColumnRenamed("churn", "label")
df_test_ml  = df_test.withColumnRenamed("churn", "label")

# ── Clasificador
rf = RandomForestClassifier(
    featuresCol="features",
    labelCol="label",
    numTrees=100,
    maxDepth=5,
    seed=42
)

# ── Pipeline: cadena de etapas
pipeline = Pipeline(stages=[indexer, encoder, assembler, scaler, rf])

# ── Entrenamiento: fit ejecuta TODOS los Estimators en orden
modelo = pipeline.fit(df_train_ml)
print(type(modelo))  # 

# ── Predicción: transform ejecuta TODOS los Transformers en orden
predicciones = modelo.transform(df_test_ml)
predicciones.select("id", "label", "prediction", "probability").show()
# +---+-----+----------+--------------------+
# | id|label|prediction|         probability|
# +---+-----+----------+--------------------+
# |  3|    1|       1.0|[0.12, 0.88]        |
# |  5|    1|       1.0|[0.09, 0.91]        |

# ── Evaluación
evaluator = BinaryClassificationEvaluator(
    labelCol="label",
    rawPredictionCol="rawPrediction",
    metricName="areaUnderROC"  # "areaUnderPR" también disponible
)
auc = evaluator.evaluate(predicciones)
print(f"AUC-ROC: {auc:.4f}")

4.3 K-Means clustering

python
from pyspark.ml.clustering import KMeans
from pyspark.ml.evaluation import ClusteringEvaluator

kmeans = KMeans(
    featuresCol="features",
    predictionCol="cluster",
    k=3,
    maxIter=20,
    seed=42
)
modelo_km = Pipeline(stages=[assembler, scaler, kmeans]).fit(datos)
clustered = modelo_km.transform(datos)

# Silhouette score
evaluator_km = ClusteringEvaluator(
    predictionCol="cluster",
    featuresCol="features",
    metricName="silhouette"
)
silhouette = evaluator_km.evaluate(clustered)
print(f"Silhouette: {silhouette:.4f}")

5. Cross-Validation y tuning de hiperparámetros

python
from pyspark.ml.tuning import CrossValidator, ParamGridBuilder

# Construir grid de hiperparámetros
param_grid = (ParamGridBuilder()
    .addGrid(rf.numTrees, [50, 100, 200])
    .addGrid(rf.maxDepth, [3, 5, 8])
    .build()  # genera 3 × 3 = 9 combinaciones
)

# CrossValidator: k-fold CV
cv = CrossValidator(
    estimator=pipeline,
    estimatorParamMaps=param_grid,
    evaluator=evaluator,
    numFolds=3,       # 3-fold cross validation
    seed=42,
    parallelism=2     # entrenar 2 modelos en paralelo (ajustar a CPUs)
)
cv_modelo = cv.fit(df_train_ml)

# Mejor modelo
mejor = cv_modelo.bestModel
print("Mejor numTrees:", mejor.stages[-1].getNumTrees)  # acceder al último stage (RF)

# Las métricas de cada fold
print(cv_modelo.avgMetrics)   # lista de AUC medios por combinación de params

6. Guardar y cargar modelos

python
# Guardar el PipelineModel completo (preproceso + modelo)
modelo.write().overwrite().save("/modelos/churn_v1")

# Guardar solo el CrossValidator model
cv_modelo.bestModel.write().overwrite().save("/modelos/churn_cv_best")

# Cargar
from pyspark.ml import PipelineModel
modelo_cargado = PipelineModel.load("/modelos/churn_v1")

# Predecir con modelo cargado (en otro proceso/sesión Spark)
nuevos = spark.createDataFrame([
    {"id": 99, "pais": "ES", "edad": 31, "ltv": 750.0, "meses": 4, "churn": 0}
])
resultado = modelo_cargado.transform(nuevos.withColumnRenamed("churn", "label"))
resultado.select("id", "prediction", "probability").show()
💡
MLlib vs scikit-learn en Spark: si tus datos caben en una sola máquina (<50GB), usa scikit-learn y Pandas — es más rápido y flexible. Usa MLlib cuando tus datos no caben en RAM o necesitas entrenar sobre datos que ya están en un cluster Spark.
