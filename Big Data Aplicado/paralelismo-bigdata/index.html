Módulo 1 — Fundamentos del paralelismo en Python

Antes de generar carga, hay que entender cómo Python ejecuta código en paralelo y cuándo realmente lo hace. El GIL, los hilos, los procesos y el event loop no son intercambiables.

Capítulo 01

El mapa mental

Concurrencia vs Paralelismo vs Asincronía. CPU-bound vs IO-bound. El diagnóstico que decide qué herramienta usar.
Capítulo 02

El GIL

Qué es el Global Interpreter Lock, qué bloquea exactamente, qué no bloquea, y cuándo te destroza el rendimiento.
Capítulo 03

threading

Hilos, Lock, RLock, Semaphore, Event, Barrier. Race conditions con ejemplo real que falla. Deadlocks y cómo diagnosticarlos.
Capítulo 04

multiprocessing

Procesos que esquivan el GIL. Pool, Process, Queue, Pipe, memoria compartida. Cuándo Pool.map vs Pool.apply_async.
Capítulo 05

concurrent.futures

La API moderna. ThreadPoolExecutor vs ProcessPoolExecutor. submit, map, as_completed. Cancelación y manejo de errores.
Capítulo 06

asyncio

Event loop, corrutinas, gather, create_task. aiohttp para 10.000 conexiones donde threading muere con 500.

Módulo 2 — Stress Testing con Python

De qué sirve un clúster si no lo puedes estresar. Locust desde el laboratorio real del workspace, y nuestro propio cliente de carga para protocolos que Locust no cubre.

Capítulo 07

Locust desde el laboratorio

El locustfile.py del workspace diseccionado línea a línea. HttpUser, @task, wait_time. Qué leer en la UI: RPS, p95, p99, fail_ratio.
Capítulo 08

Locust avanzado

SequentialTaskSet, catch_response, watchdog de fallos, modo distribuido master/worker, exportar CSV y HTML.
Capítulo 09

Nuestro propio stress tester

Cliente de carga con asyncio + aiohttp. Métricas propias p50/p95/p99. Herramienta reutilizable para Kafka, gRPC, sockets TCP.

Módulo 3 — Java/JVM: cuando Python no llega

Java no tiene GIL. Sus primitivas de concurrencia son más expresivas y su JVM aguanta más carga por proceso. Gatling puede simular 100.000 usuarios desde una sola máquina.

Capítulo 10

Threads en Java

Thread, Runnable, ExecutorService. synchronized, volatile, AtomicInteger. Productor-consumidor con BlockingQueue.
Capítulo 11

java.util.concurrent

CountDownLatch, CyclicBarrier, Semaphore. CompletableFuture encadenado. Fork/Join Framework para CPU-bound masivo.
Capítulo 12

Gatling

DSL declarativo, scenarios, feeders, throttle. HTTP y Kafka. Informe HTML profesional. 100k usuarios desde una máquina.
